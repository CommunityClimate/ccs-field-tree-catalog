Why this matters: Verra issues carbon credits based on verified planted trees. If a tree's GPS sits outside its field boundary, Verra cannot confirm it was planted in the correct place. Every tree correctly positioned within its farmer's registered field strengthens the submission.
Roadmap to 99% compliance
1
Fix overlapping field boundaries
— fields share boundary area with a neighbour. Use the Polygon Corrector to redraw so no two fields overlap. Mandatory — Verra will reject double-counted land regardless of tree accuracy.
Priority: mandatory before submission · affects carbon credit validity
2
Correct polygons for — trees >10m outside
Genuine boundary errors. Use the "✗ Needs correction" filter, sorted by most trees affected. Use the Polygon Corrector to redraw. Start with fields that have the most trees outside.
Fixing all of these moves compliance: — → —
3
Review — trees in the 5–10m zone
Check the GPS accuracy reading in Fieldforce for each tree. If the device reported accuracy >5m, document this — Verra accepts the tree. Only correct the polygon if GPS reading was good but tree is still outside.
Resolving these moves compliance: — → —
4
Leave the — tolerance trees alone
Already Verra compliant. Correcting their polygons risks pushing boundaries into neighbouring farmers' land and creating new overlaps. Document the GPS tolerance policy and present to the verifier.
